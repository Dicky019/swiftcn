# Security Policy

## Reporting a Security Issue

Please **do not** create public issues for security vulnerabilities.

Instead, report security issues by emailing **security@swiftcn.dev**.

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Your contact information

We will respond within 48 hours and work with you to understand and address the issue.

## Code Copy-Paste Disclaimer

swiftcn is designed for copying code into your project. Users are responsible for:
- Reviewing code before using it in production
- Keeping dependencies updated
- Following security best practices

We are not liable for security vulnerabilities arising from code usage.

## Security Updates

Security updates are communicated through:
- GitHub Security Advisories
- README updates
- Release notes

Please watch this repository for security announcements.
